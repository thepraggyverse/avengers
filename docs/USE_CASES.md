# Use Cases

Use these prompts as copy-paste starting points. They are written with clear leading words so routing is predictable.

| Situation | Prompt | Expected Skill |
|---|---|---|
| Start from uncertainty | `Use $a-stark-router. I have a vague product idea and need the right Avengers lens.` | `a-stark-router` |
| Build a crude first version | `Use $a-mark-one-prototype. Define the Mark 1 version I can build today.` | `a-mark-one-prototype` |
| Turn failure into an upgrade | `Use $a-mistake-to-upgrade. My launch failed; extract the upgrade rule and prevention rule.` | `a-mistake-to-upgrade` |
| Work with constraints | `Use $a-cave-resourcefulness. I have almost no budget, so inventory what already exists.` | `a-cave-resourcefulness` |
| Break permission waiting | `Use $a-permission-trap-breaker. I keep waiting for approval before starting.` | `a-permission-trap-breaker` |
| Handle too many projects | `Use $a-multi-project-reactor. Rank my active projects and find the reactor core.` | `a-multi-project-reactor` |
| Review automation risk | `Use $a-armor-around-world-check. This safety automation may over-control users.` | `a-armor-around-world-check` |
| Explain an expert idea | `Use $a-curse-of-knowledge-translator. I need to explain a risk others do not see yet.` | `a-curse-of-knowledge-translator` |
| Ground an answer in sources | `Use $a-source-grounded-synthesis. Answer from my local corpus without quoting long passages.` | `a-source-grounded-synthesis` |
| Save a reusable lesson | `Use $a-avengers-compound. Save what this run taught us as a reusable learning note.` | `a-avengers-compound` |
| Audit stale guidance | `Use $a-avengers-refresh. Check for stale docs, overlapping skills, and weak memory notes.` | `a-avengers-refresh` |
| Continue later | `Use $a-avengers-handoff. Write a concise handoff with repo state, tests, risks, and next action.` | `a-avengers-handoff` |

## Chaining Examples

### Mark 1 Sprint

```text
Use $a-stark-operating-system on this product idea.
Then use $a-mark-one-prototype for the first build.
Then use $a-mark-two-test-flight for the first stress test.
Then use $a-mistake-to-upgrade after the test result.
```

### Governance Review

```text
Use $a-armor-around-world-check on this automation.
Then use $a-ultron-failure-review for unintended consequences.
Then use $a-ethics-of-control for consent, oversight, and rollback.
```

### Compound Memory

```text
Use $a-avengers-compound after this run.
Capture request, skills used, sources consulted, result, reusable lesson, related skills, and refresh status.
```

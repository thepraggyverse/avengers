# Avengers Context

This file defines durable Avengers skill-pack vocabulary. Keep it short and practical. Add terms when they help future agents choose skills, explain outputs, or avoid confusing phrasing.

## Operating Principles

| Principle | Meaning | Related Skills |
|---|---|---|
| Mark 1 before polish | Build the smallest working proof before optimizing. | `a-mark-one-prototype`, `a-mark-two-test-flight` |
| Failure becomes upgrade | Treat mistakes as design data, not identity. | `a-mistake-to-upgrade`, `a-anti-repeat-control` |
| Constraints are materials | Use what exists by function before waiting for ideal resources. | `a-cave-resourcefulness`, `a-functional-attributes` |
| Protection needs governance | Safety systems need consent, oversight, and rollback. | `a-armor-around-world-check`, `a-ethics-of-control` |
| Learning should compound | Useful lessons should become searchable notes, context, or tests. | `a-avengers-compound`, `a-avengers-refresh` |

## Terms

| Term | Definition | Use |
|---|---|---|
| A-skill | Any `a-` prefixed Avengers skill in `skills/`. | Search/discovery shorthand across harnesses. |
| Avengers memory | The repo-local docs layer for reusable lessons, run logs, refresh reports, context, and handoffs. | Use when a result should survive the chat. |
| Corpus hook | A source file name or reference pointer used to ground a skill without committing transcript text. | Use for source-grounded synthesis. |
| Refresh status | A note's lifecycle marker: fresh, needs-refresh, merged, or stale. | Use in learning notes and refresh reports. |

## Terms To Avoid Or Clarify

| Phrase | Preferred Wording | Reason |
|---|---|---|
| Unofficial operating system | Avengers-inspired skill pack | Clearer and less grandiose. |
| Tony Stark plugin | Avengers skill/plugin pack | The pack includes leadership, MCU analysis, memory, and source-grounded work beyond one character. |
| Dump the transcripts | Synthesize from local source files | Keeps the public repo and outputs copyright-safe. |
